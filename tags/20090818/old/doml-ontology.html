Abstract

To keep the DAISY Online Delivery Specification platform agnostic, all simple (aka primitive) and complex (aka custom) types mentioned in the DAISY Online Delivery Specification Web Service API document are described using the W3C XML Schema Language. Two of the three documents that comprise the normative part of the DAISY Online Delivery Specification i.e. DAISY Online Delivery Specification Markup Language Schema and DAISY Online Delivery Specification Markup Language Ontology (this document) describe all these types in a machine readable (xsd) and a human readable (plain english) form respectively.

This document is normative. It lays out all stipulations pertaining to the Lexical and Value spaces of all the types in the DAISY Online Delivery Specification Markup Language.

This document is a part of a suite of three documents that together compose the normative part of the DAISY Online Delivery Specification. All these documents are detailed in the DAISY Online Delivery Specification Overview.

1. Introduction

This section is informative.

The DAISY Online Delivery Specification Markup Language is an XML vocabulary. The namespace for this vocabulary is http://www.daisy.org/ns/doml#. All types detailed in this document together form this vocabulary. Henceforth, the DAISY Online Delivery Specification Markup Language is referred to as DOML in this document.

To aid understanding, the types in doml are categorised into five distinct categories that roughly resemble the Functional categories in the DAISY Online Delivery Specification Web Service API document.

Please note, a type in the context of this document is also an XML element. Therefore, the word type and element are used interchangeably throughout this document.

It is envisaged that native programming constructs will be generated from the types detailed in this document. These native programming constructs can also be directly infered by an inferring engine through the DOML Schema document.

The key words "MUST", "MUST NOT", "REQUIRED", "SHALL", "SHALL NOT", "SHOULD", "SHOULD NOT", "RECOMMENDED", "MAY", and "OPTIONAL" in this document are to be interpreted as described in RFC 2119.

The ensuing sections are an ontological elaboration of all types in DOML.

1.1. Namespaces and the DAISY Online Delivery Specification Markup Language

This section is normative.

All types in the DAISY Online Delivery Specification Markup Language MUST incorporate support for the Namespaces in XML 1.0 W3C standard by adhering to the following rules:

  1. An xmlns attribute MUST be present on all permitted root elements.
  2. The value of the xmlns attribute MUST be http://www.daisy.org/ns/doml#.
  3. All elements that are not a part of http://www.daisy.org/ns/doml#, declared as the default namespace through the xmlns attribute, MUST be prefixed with a prefix bound to the particular namespace e.g. xmlns:foo="...." and then <foo:element1></foo:element1>
  4. A compliant Service or Reading System MUST ignore any prefixed elements or attributes that it does not understand.

2. General types

This section is informative.

The General category contains types that can be used to deliver the following semantic information:

  • Information pertaining to a user's settings on a Reading System.
  • Information pertaining to the specific characteristics of a Reading System.
  • Information pertaining to the nature of content that MUST be rendered on a Reading System.
  • Information pertaining to an Announcement.
  • Metadata information pertaining to an Announcement.

A detailed description of each of these general purpose types follows.

2.1. The settings type

This section is normative.

The settings type is used by the getSettings and setSettings functions in the DAISY Online Delivery Specification Web Service API.

Attributes of the settings type (aka element)

This element has no attributes.

Content options for the settings type (aka element)

The settings element MAY contain the following elements:

  • Skippability - This element contains the settings specific to skippability. To find out more about skippability in ANSI/NISO Z39.86-2005, please visit http://www.daisy.org/z3986/2005/Z3986-2005.html#custtest This is an OPTIONAL element that MAY occur once inside a settings element. It has no attributes. It MUST contain the following elements:
    • bookStruct - This element is an OPTIONAL element that MAY occur one or more times inside the skippability element. It MUST contain the following attributes:
      • type - The data type of this attribute is String. The allowed values for this attribute are: PAGE_NUMBER | NOTE | NOTE_REFERENCE | ANNOTATION | LINE_NUMBER | OPTIONAL_SIDEBAR | OPTIONAL_PRODUCER_NOTE.
      • skippable - The value of this attribute indicates whether the element specified by the type attribute is skippable. The data type of this element is Boolean.
  • uiVolume - This element contains a value that represents the volume setting for the User Interface of the Reading System. The data type of this element is Int. The allowed values are from 1 to 100. This element is an OPTIONAL element that MAY only occur once inside the settings element.
  • contentVolume - This element contains a value that represents the volume setting for the Content reading mode of the Reading System. The data type of this element is Int. The allowed values are from 1 to 100. This element is an OPTIONAL element that MAY only occur once inside the settings element.
  • uiTone - This element contains a value that represents the tone setting for the User Interface of the Reading System. The data type of this element is Int. The allowed values are from 1 to 100. This element is an OPTIONAL element that MAY only occur once inside the settings element.
  • contentTone - This element contains a value that represents the tone setting for the Content reading mode of the Reading System. The data type of this element is Int. The allowed values are from 1 to 100. This element is an OPTIONAL element that MAY only occur once inside the settings element.
  • uiSpeed - This element contains a value that represents the speed setting for the User Interface of the Reading System. The data type of this element is Int. The allowed values are from 1 to 100. This element is an OPTIONAL element that MAY only occur once inside the settings element.
  • contentSpeed - This element contains a value that represents the speed setting for the Content reading mode of the Reading System. The data type of this element is Int. The allowed values are from 1 to 100. This element is an OPTIONAL element that MAY only occur once inside the settings element.
  • uiPreferredLanguage - This element contains a value that represents the preferred language setting for the User Interface of the Reading System. The data type of this element is Language. This element is an OPTIONAL element that MAY only occur once inside the settings element.
  • contentPreferredLanguage - This element contains a value that represents the preferred language setting for the Content reading mode of the Reading System. The data type of this element is Language. This element is an OPTIONAL element that MAY only occur once inside the settings element.
  • One or more elements from any other namespace. Please note, no other elements from the doml namespace (http://www.daisy.org/ns/doml#) are permitted here.

2.2. The readingSystemAttributes type

This section is normative.

The readingSystemAttributes type is used by the setReadingSystemAttributes function in the DAISY Online Delivery Specification Web Service API.

This type MUST be used to deliver the specific characteristics (aka attributes) of the Reading System to the Service.

Attributes of the readingSystemAttributes type (aka element)

This element has no attributes.

Content options for the readingSystemAttributes type (aka element)

The readingSystemAttributes type MAY contain the following elements:

  • One or more elements from any other namespace. Please note, no other elements from the doml namespace (http://www.daisy.org/ns/doml#) are permitted here.
  • manufacturer - This element holds the information about the manufacturer of the Reading System. The data type of this element is String. This element MUST occur only once inside the readingSystemAttributes element. It is a REQUIRED element. There are no restrictions placed in DOML Schema on the value space of this element. To find more information about the pattern that the value of this element might adhere to, refer to the DAISY Consortium Website.
  • model - This element holds the information about the model of the Reading System. It is a REQUIRED element. The data type of this element is String. It MUST occur only once inside the readingSystemAttributes element.
  • serialNumber - This element holds the information about the serialNumber of the Reading System. It is a REQUIRED element. The data type of this element is String. It MUST occur only once inside the readingSystemAttributes element.
  • firmwareVersion - This element holds the information about the firmware version of the Reading System. It is a REQUIRED element. The data type of this element is String. It MUST occur only once inside the readingSystemAttributes element.
  • config - This element holds various pieces of configuration information about the Reading System. It is a REQUIRED element. The data type of this element is String. It MUST occur only once inside the readingSystemAttributes element. This element MUST have the following elements:
    • supportsEmbeddedAudio - This element indicates whether the Reading System supports embedded audio in the message element. This element is a REQUIRED element that MUST occur only once inside the config element. The data type of this element is Boolean.
    • supportsMultipleSelections - This element indicates whether the Reading System supports multiple selections for a choices type question. This is a REQUIRED element that MUST occur only once inside the config element. The data type of this element is Boolean. If this element is set to false, the Service MUST not present the allowMultipleSelections and multipleSelectionsCallbackFunction attributes on the userChoices element to the Reading System.
    • supportsGroupedQuestions - This element indicates whether the Reading System supports the rendering of a grouped set of questions. This element is a REQUIRED element that MUST occur only once inside a config element. The data type of this element is Boolean. If this element is set to false, the Service MUST not specify the value getGroupedQuestions for any callbackFunction or multipleSelectionsCallbackFunction elements inside any type.
    • protocolVersion - This element contains the latest supported version of the DAISY Online Delivery Specification on the Reading System. This is a REQUIRED element that MUST only occur once inside the config element. The data type of this element is String.
    • supportedContentStandards - This element contains information about the supported content standards on the Reading System. It is a REQUIRED element that MUST occur once inside a config element. This element MUST contain the following element:
      • supportedContentStandard - This element contains information about a single supported content standard on the Reading System. This element is a REQUIRED element that MUST occur one or more times inside the supportedContentStandards element. The data type of this element is String. It is important to note that the DAISY Online Delivery Specification does not place any restrictions on the value space of this element.
    • The config element MUST contain a single instance of a mimeTypes element.
    • inputTypes - This element is a REQUIRED element that MUST occur only once inside the config element. It has no attributes. It MUST contain the following element:
      • input - This element is an empty element. It is a REQUIRED element that MUST occur one or more times inside the inputTypes element. It MUST have the following attributes:
        • type - This attribute contains information about the mode of input being represented by the input element. the data type of this attribute is String. The allowed values for this attribute are: audio, text, braille, video and image. This attribute is a REQUIRED attribute.
        • brailleCode - This attribute contains the braille code that MUST be used for the braille input to the question. the data type of this attribute is String. This attribute MUST have unicode as its value; no other value is allowed. The unicode braille characters are in position u2800 to u28ff and are described at http://unicode.org/charts/PDF/U2800.pdf.
        • xml:lang - This attribute holds the value of the language in which the input will be received. This attribute is a part of the XML namespace. This is an OPTIONAL attribute. If omitted, the Service Provider MUST make their best choice of language to be used.
    • availableMemory - This element contains the information in bytes about the memory available on a Reading System. This element is a REQUIRED element that MUST occur only once inside a config element. The data type of this element is Int. It is important to note that available memory in this context is the size in bytes of the largest piece of content that MUST be downloaded and stored on the Reading System.
    • modes - This element contains information about the content access modes supported on a Reading System. It is a REQUIRED element that MUST occur only once inside a config element. It MUST contain the following element:
      • mode - This element contains information about a single supported mode on the Reading System. This element is a REQUIRED element that MUST occur one or more times inside a modes element. The data type of this element is String. The allowed values for this element are: download and stream.
    • supportedProtocols - This element contains information about the protocols supported on a Reading System for content access. It is a REQUIRED element that MUST occur only once inside a config element. It MUST contain the following element:
      • supportedProtocol - This element contains information about a single supported protocol on a Reading System. This element is a REQUIRED element that MUST occur one or more times inside a supportedProtocols element. The data type of this element is String. The DAISY Online Delivery Specification does not place any restriction on the value space of this element. It is important to note that http and https do not need to be explicitly declared as supported protocols through this element; both http and https MUST be supported by all Reading Systems to claim Compliance to the DAISY Online Delivery Specification.
    • preferredLanguage - This element contains information about the preferred language on the Reading System. This element is a REQUIRED element that MUST occur only once inside a config element. The data type of this element is Language.
    • bandwidth - This element contains information in kilobits per second about the bandwidth available to the Reading System. This element is a REQUIRED element that MUST only occur once inside a config element. The data type of this element is Int.

2.3. The mimeTypes type

This section is normative.

The mimeTypes type is used by the setMessageMimeTypes function in the DAISY Online Delivery Specification Web Service API.

This type is also a child type of the readingSystemAttributes type. This type MUST be used when information about the Mime types supported in a message type needs to be delivered to the Service.

Attributes of the mimeTypes type (aka element)

This element has no attributes.

Content options for the mimeTypes type (aka element)

The mimeTypes type MAY contain the following elements:

  • mimeType - This element is an empty element. It is a REQUIRED element that MUST occur one or more times inside the mimeTypes element. It MUST have the following attributes:
    • type - This attribute contains information about a single MIME type being represented by the mimeType element. the data type of this attribute is String. It is important to note that the DAISY Online Delivery Specification does not place any restrictions on the value space of this attribute. For more information about the MIME types that MAY be used as a value of this attribute, please visit IANA MIME Media Types. This attribute is a REQUIRED attribute.
    • brailleCode - This attribute contains the braille code that MUST be used for rendering braille. The data type of this attribute is String. This attribute MUST have unicode as its value; no other value is allowed. The unicode braille characters are in position u2800 to u28ff and are described at http://unicode.org/charts/PDF/U2800.pdf.
    • xml:lang - This attribute holds the value of the language in which the content will be rendered. This attribute is a part of the XML namespace. This is an OPTIONAL attribute. If omitted, the Service Provider MUST make their best choice of language to be used.

2.4. The announcements type

This section is normative.

The announcements type is used by the getServiceAnnouncements and getUserAnnouncement functions in the DAISY Online Delivery Specification Web Service API.

This type MUST be used to deliver announcements from a Service to a Reading System.

Attributes of the announcements type (aka element)

This element has no attributes.

Content options for the announcements type (aka element)

The announcements type MAY contain the following elements:

  • announcement - This element is an OPTIONAL element that MAY occur one or more times inside the announcements element. It MUST contain one or more message elements. It MUST have the following atributes:
    • id - This attribute contains a unique identifier for this announcement element. The data type of this atribute is String. This attribute is a REQUIRED attribute.
    • sequenceNumber - The value of this attribute is the ordinal of This element in the encapsulating fragment. The data type of this atribute is int. This attribute is a REQUIRED attribute.
    • type - This attribute holds a value that indicates the nature of this announcement element. The data type of this attribute is String. The allowed values for this attribute are: warning, error, information and system. It is an OPTIONAL attribute. It defaults to information.
    • priority - This attribute holds a value that indicates the priority of this announcement element. The data type of this attribute is Int. The allowed values are 1 to 3, where 1 denotes the highest priority and 3 the lowest. It is an OPTIONAL attribute. It defaults to 3.

2.5. The announcementsMetadata type

This section is normative.

The announcementsMetadata type is used by the getServiceAnnouncementsMetadata function in the DAISY Online Delivery Specification Web Service API.

This type MUST be used to deliver metadata information about Service announcements. It is anticipated that the announcements type MAY be quite large if audio files are encapsulated in it, and thus the size information available in this type will enable a Reading System to request individual or a series of announcement elements in an announcements element.

Attributes of the announcementsMetadata type (aka element)

This element has no attributes.

Content options for the announcementsMetadata type (aka element)

The announcementsMetadata type MAY contain the following elements:

  • announcementMetadata - This element contains metadata information about an announcement element inside an announcements element. It is an empty element. It is an OPTIONAL element that MAY occur one or more times inside the announcementsMetadata element. It MUST have the following attributes:
    • size - This attribute holds the size in bytes of the announcement element inside an announcements element. The data type of this attribute is Int. It is a REQUIRED attribute.
    • sequenceNumber - The value of this attribute is the ordinal of This element in the encapsulating fragment. This value corresponds to the value contained in the sequenceNumber attribute of the announcement.announcements element. The data type of this atribute is int. This attribute is a REQUIRED attribute.

2.6. The message type

This section is normative.

The message type is not used directly by any functions in the DAISY Online Delivery Specification Web Service API. This type is a child type of other types in DOML.

The DAISY Online Delivery Specification provides three options to the Service for delivering audio files to a Reading System through this element:

  • Option 1: The audio file MAY be delivered over http or https - to use this option, a Service MUST set the useWebService attribute to false, and src attribute to the uri of the audio file on the audio element of this element.
  • Option 2: The audio file MAY be delivered via the getResource web service function - To use this option, a Service MUST set the useWebService attribute to true, and the src attribute to the uri of the audio file on the audio element of this element.
  • Option 3: The audio file MAY be delivered by encapsulating it in this type - To use this option, a Service MUST set the useWebService attribute to false, the src attribute to an empty string or omit the src attribute altogether, and embed the audio file in the data element of the audio element of this element. It SHOULD be noted that the reading system will not be able to start rendering the audio file embedded in the data element of the audio element of this element, until the entire SOAP message has been received and disassembled. Therefore it is recommended that this option be used only to embed small audio files into the data element of the audio element of this element. on the other hand, when determining whether or not to use this option, the Service designer SHOULD balance the benefit of compressing the audio files before embedding them into a single SOAP message against the total number of audio files to be accessed, and the cumulative delay caused by network latency when the reading system MUST access each audio file in turn.

Attributes of the message type (aka element)

This element MAY have the following attribute:

  • xml:lang - This attribute holds a value that indicates the language of this element. This attribute is a part of the XML namespace. This is an OPTIONAL attribute. If omitted, the Service Provider MUST make their best choice of language to be used.

Content options for the message type (aka element)

The message type MAY contain the following elements:

  • text - This element contains the text portion of the message element. It is an OPTIONAL element that MAY occur once inside the message element. The data type of this element is String. This element has no attributes.
  • audio - This element contains the audio portion of the message element. This element MUST contain one instance of a data element that holds the embedded audio file if the third option detailed in the introduction to the message type for delivering audio files is used. The data type of the data element is Base64Binary. The data element MUST have one REQUIRED attribute: codec. The data type of the codec attribute is String. There are no explicit restrictions on the values that the codec attribute MUST contain. to find more information about what values MUST be used with this attribute, please refer to the RFC 4281 document. This element is an OPTIONAL element and MAY occur only once inside the message element. This element MUST have the following attributes:
    • size - This attribute contains the size in bytes of the audio file. The data type of this attribute is Int. It is a REQUIRED attribute.
    • src - This attribute contains the uri to the audio file. If this attribute contains an empty string value or is omitted altogether, the audio file MUST be embedded in the data element of the audio element. The data type of this attribute is String. It is an OPTIONAL attribute.
    • useWebService - This attribute indicates whether the audio file is to be retrieved via the getResource function in the DAISY Online Delivery Specification Web Service API (see option 2 in the introduction to the message type). The data type of this attribute is Boolean. It is a REQUIRED attribute.
    • rangeBegin - This attribute holds the value in bytes of the start byte of the byte range that is the audio portion of the message element. The data type of this attribute is Int. It is an OPTIONAL attribute.
    • rangeEnd - This attribute holds the value in bytes of the end byte of the byte range that is the audio portion of the message element. The data type of this attribute is Int. It is an OPTIONAL attribute.

3. Content types

This section is informative.

The content category contains types that can be used to deliver the following semantic information:

  • Information pertaining to a search criteria.
  • Information pertaining to the status of the specified content on the Reading System.
  • Information pertaining to the metadata for the specified content.
  • Information pertaining to a list of content.
  • Metadata information pertaining to a list of content.

A detailed description of each of these content types follows.

3.1. The SearchCriteria type

This section is normative.

The searchCriteria type is used by the searchContent function in the DAISY Online Delivery Specification Web Service API.

This type MUST be used when a Reading System needs to specify a search criteria for a search.

Attributes of the searchCriteria type (aka element)

This element has no attributes.

Content options for the searchCriteria type (aka element)

The searchCriteria type MAY contain the following elements:

  • title - the value of this element is the title of the publication. the data type of this element is String. It is an OPTIONAL element and MAY occur only once inside the searchCriteria element. It has no attributes.
  • size - The value of this element is the size of the publication in bytes. The data type of this element is Int. It is an OPTIONAL element and MAY occur only once inside the searchCriteria element. It has no attributes.
  • author - The value of this element is the author of the publication. The data type of this element is String. It is an OPTIONAL element and MAY occur only once inside the searchCriteria element. It has no attributes.
  • genre - The value of this element is the genre of the publication. The data type of this element is String. It is an OPTIONAL element and MAY occur only once inside the searchCriteria element. It has no attributes.
  • publisher - The value of this element is the publisher of the publication. The data type of this element is String. It is an OPTIONAL element and MAY occur only once inside the searchCriteria element. It has no attributes.
  • narrator - The value of this element is the narrator of the publication. The data type of this element is String. It is an OPTIONAL element and MAY occur only once inside the searchCriteria element. It has no attributes.
  • language - The value of this element is the language of the publication. The data type of this element is String. It is an OPTIONAL element and MAY occur only once inside the searchCriteria element. It has no attributes.
  • issueDate - The value of this element is the issue date of the publication. The issue date is the date when the publication was added to the catalog for the client. The data type of this element is String. The constraint on the value set for this element is the ISO 8601 date pattern. It is an OPTIONAL element and MAY occur only once inside the searchCriteria element. It has no attributes.
  • sortBy - The value of this element is the filter by which the matching publications will be sorted as a result of the search. The data type of this element is String. The allowed values for this element are: title, size, author, genre, publisher, narrator, language and issueDate i.e. all other child elements of the searchCriteria element. It is an OPTIONAL element and MAY occur only once inside the searchCriteria element. It MUST have the following attribute:
    • type - This attribute holds a value that indicates whether the filter is to be applied in an ascending or descending order. The data type of this attribute is String. the only two allowed values are ascending or descending. It is a REQUIRED attribute.

3.2. The contentStatus type

This section is normative.

The contentStatus type is used by the getContentStatus and setContentStatus functions in the DAISY Online Delivery Specification Web Service API.

Attributes of the contentStatus type (aka element)

This element has no attributes.

Content options for the contentStatus type (aka element)

The contentStatus type MAY contain the following elements:

  • publication - This element MUST occur one or more times inside the contentStatus element. This element MUST have the following attribute:
    • id - This attribute contains the unique id of the publication represented by the publication element. The data type of this attribute is String. It is a REQUIRED attribute.
    The publication element MUST contain the following elements:
    • status - the value of this element is the status of the publication on the Reading System. The data type of this element is String. The allowed values for this element are: downloading, downloaded, pending and streaming. It is a REQUIRED element that MUST occur only once inside a publication element. It has no attributes.
    • downloadedBytes - the value of this element is the total downloaded size in bytes of the publication on the Reading System. The data type of this element is Int. It is an OPTIONAL element that MUST occur only once inside a publication element. It has no attributes.
    • storageType - the value of this element is the location of the publication on the Reading System. The data type of this element is String. The allowed values for this element are: internal, removable and network. It is a REQUIRED element that MUST occur only once inside a publication element. It has no attributes.

3.3. The contentMetadata type

This section is normative.

The contentMetadata type is used by the getContentMetadata function in the DAISY Online Delivery Specification Web Service API.

This type MUST be used when the Service needs to deliver metadata information about the specified content to the Reading System.

Attributes of the contentMetadata type (aka element)

This element MAY have the following attributes:

  • id - This attribute holds the unique identifier of the content that the metadata is for. The data type of the id attribute is String. It is a REQUIRED attribute.
  • type - This attribute holds the information about the type of content that the metadata is for. The data type of the type attribute is String. the allowed values for this attribute are: book, magazine, newspaper and other. It is an OPTIONAL attribute.
  • accessMode - This attribute holds the information about the modes of access for the content that the metadata is for. The data type of the accessMode attribute is String. The allowed values for this attribute are: stream, download and unrestricted. If this attribute contains an empty string value, the Reading System MUST infer the value to be unrestricted. It is a REQUIRED attribute.
  • useWebService - This attribute indicates whether the getResource function in the DAISY Online Delivery Specification Web Service API SHOULD be used to access the content that the metadata is for. The data type of the useWebService attribute is Boolean. It is a REQUIRED attribute.

Content options for the contentMetadata type (aka element)

The contentMetadata type MAY contain the following elements:

  • sample - This element is an OPTIONAL element that MAY occur only once inside the contentMetadata element. it MUST contain one message element.
  • contentStandard - This element is a REQUIRED element that MUST occur only once inside the contentMetadata element. The value contained in this element is the specific standard that the content being described by the contentMetadata fragment adheres to e.g. DAISY/NISO 2002. The value contained in this element MUST be a value specified by the Reading System as a supported content standard through the supportedContentStandard.supportedContentStandards.config.readingSystemAttributes element.
  • provider - This element is an empty element. It is an OPTIONAL element that MAY occur only once inside the contentMetadata element. It MUST have the following attribute:
    • id - This attribute holds a unique identifier for the provider represented by the provider element. The data type of this attribute is String. To find more information about the possible unique identifier values for a provider, refer to the DAISY Consortium Website. this is a REQUIRED attribute.
  • service - This element is an empty element. It is an OPTIONAL element that MAY occur only once inside the contentMetadata element. It MUST have the following attribute:
    • id - This attribute holds a unique identifier for the service represented by the service element. The data type of this attribute is String. To find more information about the possible unique identifier values for a service, refer to the DAISY Consortium Website. this is a REQUIRED attribute.
  • metadata - This element is an OPTIONAL element that MAY only occur once inside the contentMetadata element. It has no attributes. It MUST have the following elements:
    • title - the value of this element is the title of the content. the data type of this element is String. It is a REQUIRED element and MUST occur only once inside the metadata element. It has no attributes.
    • size - The value of this element is the size of the content in bytes. The data type of this element is Int. It is a REQUIRED element and MUST occur only once inside the metadata element. It has no attributes.
    • author - The value of this element is the author of the content. The data type of this element is String. It is an OPTIONAL element and MAY occur only once inside the metadata element. It has no attributes.
    • genre - The value of this element is the genre of the content. The data type of this element is String. It is an OPTIONAL element and MAY occur only once inside the metadata element. It has no attributes.
    • publisher - The value of this element is the publisher of the content. The data type of this element is String. It is an OPTIONAL element and MAY occur only once inside the metadata element. It has no attributes.
    • narrator - The value of this element is the narrator of the content. The data type of this element is String. It is an OPTIONAL element and MAY occur only once inside the metadata element. It has no attributes.
    • language - The value of this element is the language of the content. The data type of this element is String. It is an OPTIONAL element and MAY occur only once inside the metadata element. It has no attributes.
    • description - The value of this element is the description of the content. The data type of this element is String. It is an OPTIONAL element and MAY occur only once inside the metadata element. It has no attributes.
    • One or more elements from any other namespace. Please note, no other elements from the doml namespace (http://www.daisy.org/ns/doml#) are permitted here.
  • resources - This element is a REQUIRED element and MUST occur only once inside the contentMetadata element. It has no attributes. It MUST contain the following elements:
    • resource - This element is a REQUIRED element that MUST occur one or more times inside the resources element. It is an empty element. It MUST have the following attributes:
      • uri - This attribute contains the path to the resource. The data type of this attribute is String. It is a REQUIRED attribute.
      • mimeType - This attribute contains the MIME type of the resource. The data type of this attribute is String. It is a REQUIRED attribute. It is important to note that the DAISY Online Delivery Specification does not place any restrictions on the value space of this attribute. For more information about the MIME types that MAY be used as a value of this attribute, please visit IANA MIME Media Types.
      • size - This attribute contains the size in bytes of the resource. The data type of this attribute is Int. It is a REQUIRED attribute.
      • localFileName - This attribute contains the local file name of the resource. The data type of this attribute is String. It is a REQUIRED attribute. The value of this attribute is the physical name of the file that this resource element represents.

5. Dynamic Menu Generation types

This section is informative.

The Dynamic Menu Generation category contains types that can be used to deliver the following semantic information:

  • Information pertaining to a input type question.
  • Information pertaining to a choices type question.
  • Metadata information pertaining to a choices type question.
  • Information pertaining to a grouped set of questions.
  • Metadata information pertaining to a grouped set of questions.

A detailed description of each of these Dynamic Menu Generation types follows.

5.1. The userInput type

This section is normative.

The userInput type is used by the getUserInput function in the DAISY Online Delivery Specification Web Service API.

This type MUST be used when a Service needs to deliver a text type question to the Reading System.

Attributes of the userInput type (aka element)

This element MAY have the following attribute:

  • id - This attribute contains the unique identifier of the text type question represented by this element. The data type of this attribute is String. This is a REQUIRED attribute.

Content options for the userInput type (aka element)

The userInput element MAY contain the following elements:

  • inputTypes - This element is an OPTIONAL element that MAY occur only once inside the userInput element. It has no attributes. It MUST contain the following element:
    • input - This element is an empty element. It is a REQUIRED element that MUST occur one or more times inside the inputTypes element. It MUST have the following attributes:
      • type - This attribute contains information about the mode of input being represented by the input element. the data type of this attribute is String. The allowed values for this attribute are: audio, text, braille, video and image. This attribute is a REQUIRED attribute.
      • brailleCode - This attribute contains the braille code that MUST be used for the braille input to the question. the data type of this attribute is String. This attribute MUST have unicode as its value; no other value is allowed. The unicode braille characters are in position u2800 to u28ff and are described at http://unicode.org/charts/PDF/U2800.pdf.
      • xml:lang - This attribute holds the value of the language that the text type question, represented by the encapsulating userInput element MUST be delivered to the Reading System in. This attribute is a part of the XML namespace. This is an OPTIONAL attribute. If omitted, the Service Provider MUST make their best choice of language to be used.
  • The userInput element MUST contain one or more message elements.
  • callbackFunction - this element contains the name of the next function to call in the DAISY Online Delivery Specification Web Service API. The data type of this element is String. The allowed values for this element are: getUserChoices, getUserAnnouncement, getUserInput, getGroupedQuestions, postUserResponses and getContentList. This element is a REQUIRED element.

5.2. The userChoices type

This section is normative.

The userChoices type is used by the getUserChoices function in the DAISY Online Delivery Specification Web Service API.

This type MUST be used when the Service needs to deliver a choices type question to the Reading System.

Attributes of the userChoices type (aka element)

This element MAY have the following attributes:

  • id - This attribute contains the unique identifier for the multiple choice question represented by the userChoices fragment. The data type of this attribute is String. This is a REQUIRED attribute.
  • allowMultipleSelections - This attribute indicates whether the choices type question being presented to the Reading System MUST have multiple answers. The data type of this attribute is Boolean. This is an OPTIONAL attribute. This attribute MUST only be presented to the Reading System if the Reading System set the supportsMultipleSelections element to true in the readingSystemAttributes element before passing it to the Service by a call to the setReadingSystemAttributes function in the DAISY Online Delivery Specification Web Service API.
  • multipleSelectionsCallbackFunction - This attribute holds the name of the next function to call in the DAISY Online Delivery Specification Web Service API. The data type of this attribute is String. If the allowMultipleSelections attribute is set to true, the value of this attribute MUST override the value of all other callbackFunction elements in the userChoices element. Furthermore, if the allowMultipleSelections attribute is true, this attribute MUST be present and contain an allowed value. the allowed values for this attribute are: getUserChoices, getUserInput, getUserAnnouncement, getGroupedQuestions, postUserResponses and getContentList. This attribute is an OPTIONAL attribute.

Content options for the userChoices type (aka element)

The userChoices type MAY contain the following elements:

  • A single instance of a message element. message is a REQUIRED element.
  • choices - This element is a REQUIRED element. It has no attributes. It MUST only occur once inside the userChoices element. It MUST contain the following elements:
    • choice - this element is a REQUIRED element. It MUST have the following attributes:
    • id - this attribute contains the unique identifier of the choice being presented to the Reading System. The data type of this attribute is String. This is a REQUIRED attribute.
    • sequenceNumber - The value of this attribute is the ordinal of This element in the encapsulating fragment. The data type of this atribute is int. This attribute is a REQUIRED attribute.
    The choice element MUST contain the following elements:
    • A single instance of a REQUIRED message element.
    • callbackFunction - this element contains the name of the next function to call in the DAISY Online Delivery Specification Web Service API. The data type of this element is String The allowed values for this element are: getUserChoices, getUserAnnouncement, getUserInput, getGroupedQuestions, postUserResponses and getContentList. This element is a REQUIRED element. If the allowMultipleSelections attribute is set to true on the userChoices element and there is a valid value in the multipleSelectionsCallbackFunction attribute, the Service MUST choose to not specify any value for this attribute.

5.3. The userChoicesMetadata type

This section is normative.

The userChoicesMetadata type is used by the getUserChoicesMetadata function in the DAISY Online Delivery Specification Web Service API.

This type MUST be used by the Service to deliver metadata information about a userChoices type to the Reading System.

Attributes of the userChoicesMetadata type (aka element)

This element has no attributes.

Content options for the userChoicesMetadata type (aka element)

The userChoicesMetadata type MAY contain the following elements:

  • userChoiceMetadata - This element is an empty element. It is a REQUIRED element that MUST occur one or more times inside the userChoicesMetadata element. It MUST have the following attributes:
    • sequenceNumber - This attribute contains a value that corresponds to the value contained in the sequenceNumber attribute of the choice element in the choices element contained inside the userChoices element. The userChoiceMetadata element provides metadata information about this choice element. The data type of this attribute is Int. It is a REQUIRED attribute.
    • size - This attribute holds the size in bytes of the choice element in the choices element of the userChoices element. The data type of this attribute is Int. It is a REQUIRED attribute.

5.4. The groupedQuestions type

This section is normative.

The groupedQuestions type is used by the getGroupedQuestionsfunction in the DAISY Online Delivery Specification Web Service API.

This type MUST be used when the Service needs to deliver a grouped set of questions to the Reading System.

Attributes of the groupedQuestions type (aka element)

This element MAY have the following attribute:

  • groupedQuestionsCallbackFunction - this attribute contains the name of the next function to call in the DAISY Online Delivery Specification Web Service API. The data type of this element is String The allowed values for this element are: getUserChoices, getUserAnnouncement, getUserInput, getGroupedQuestions, postUserResponses and getContentList. The value of this attribute MUST in all cases override the value of all callbackFunction elements inside a groupedQuestions element. The Service MAY choose to specify an empty value for all callbackFunction elements in the groupedQuestions element. This attribute is a REQUIRED attribute.

Content options for the groupedQuestions type (aka element)

The groupedQuestions type MAY contain the following element:

  • question - This element is a REQUIRED element that MUST occur one or more times inside a groupedQuestions element. This element MUST contain the following attribute:
    • sequenceNumber - The value of this attribute is the ordinal of This element in the encapsulating fragment. The data type of this atribute is int. This attribute is a REQUIRED attribute.
    This element MUST contain the following elements:
    • A single instance of a userChoices element.
    • Or a single instance of a userInput element.
    • Or a single instance of a announcements element.

5.5. The groupedQuestionsMetadata type

This section is normative.

The groupedQuestionsMetadata type is used by the getGroupedQuestionsMetadata function in the DAISY Online Delivery Specification Web Service API.

this type MUST be used when the Service needs to deliver metadata information about a groupedQuestions type to the Reading System.

Attributes of the groupedQuestionsMetadata type (aka element)

This element has no attributes.

Content options for the groupedQuestionsMetadata type (aka element)

The groupedQuestionsMetadata type MAY contain the following elements:

  • questionMetadata - This element is an empty element. It is a REQUIRED element that MUST occur one or more times inside the groupedQuestionsMetadata element. This element provides metadata information about a question.groupedQuestions element. This element MUST have the following attributes:
    • sequenceNumber - The value of this attribute corresponds to the value contained in the sequenceNumber attribute of the question.groupedQuestions element. The data type of this attribute is Int. It is a REQUIRED attribute.
    • size - This attribute contains the size in bytes of either a userChoices.question.groupedQuestions element, announcements.question.groupedQuestions element or a userInput.question.groupedQuestions element. The data type of this attribute is Int. This attribute is a REQUIRED attribute.

5.6. The userResponses type

This section is normative.

The userResponses type is used by the getUserChoices, getUserChoicesMetadata, getUserInput, getUserAnnouncement, getGroupedQuestions, getGroupedQuestionsMetadata, getContentList and getContentListMetadata functions in the DAISY Online Delivery Specification Web Service API.

This type MUST be used to deliver a User's response(s) to the Service.

Attributes of the userResponses type (aka element)

This element has no attributes.

Content options for the userResponses type (aka element)

The userResponses element MAY contain the following element:

  • userResponse - This element represents a single response from the user. It is an empty element. It is a REQUIRED element that MUST occur one or more times inside the userResponses element. This element MUST have the following attributes:
    • id - This attribute contains the unique identifier of the question that this response is for. The data type of this attribute is String. It is a REQUIRED attribute. Please note, there are two special values "home" and "back" which when specified as a value of this attribute, require the type attribute to have the value "choice". These two values are special purpose values that signify a request for the first question and previous question respectively, in the series of questions issued to the Reading System by the Service via the Dynamic Menu Generation Functions.
    • type - This attribute holds the type of question that this response is for. The data type of this attribute is String. The allowed values for this attribute are: choice, text and audio. It is a REQUIRED attribute.
    • value - This attribute holds the value of the response to the question represented by the userResponse element. The data type of this value is String. It is a REQUIRED attribute. This attribute MUST be set to the value contained in the id attribute of the choice.choices.userChoices element, if the type attribute has the value choice. If the type attribute has the value text, the value attribute MUST contain the text of the response.
    • data - This attribute contains the base 64 encoded string representation of the audio response, if the type attribute is set to audio. The data type of this value is Base64Binary. This is an OPTIONAL attribute that MUST occur if the type attribute is set to audio.

6. Exception types

This section is informative.

The Exceptions category contains types that can be used to deliver the following semantic information:

  • Information pertaining to a Not Authorised exception.
  • Information pertaining to an Internal Server Error exception.
  • Information pertaining to an Invalid Parameter exception.
  • Information pertaining to a Not Issued exception.
  • Information pertaining to a Service Busy exception.

It is envisaged that each of these exception types will be incorporated in the details element of the fault element in the SOAP message. The faultcode element of the fault element of the SOAP message SHOULD always be set to SOAP:server when incorporating any of these exception types in the aforementioned details element. There are no further restrictions on the other elements of the fault element in the SOAP message.

A detailed description of each of these exception types follows.

6.1. The notAuthorised type

This section is normative.

This type MUST be incorporated in the details element of the fault element in the SOAP message when the Service deems that the Reading System is not authorised to make the particular function call.

Attributes of the notAuthorised type (aka element)

This element has no attributes.

Content options for the notAuthorised type (aka element)

The notAuthorised element MAY contain the following elements:

  • exceptionMessage - This element contains the specific message associated to this exception type. This element is a REQUIRED element that MUST only occur once inside the notAuthorised element. The data type of this element is String. The only allowed value for this element is: This function call is not authorised.
  • exceptionTrace - This element contains the specific stack trace associated to this exception type. This element is a REQUIRED element that MUST only occur once inside the notAuthorised element. The data type of this element is String. There are no restrictions on the lexical or value spaces of this element. The Service MUST however incorporate the specific stack trace in the value of this element.

6.2. The internalServerError type

This section is normative.

This type MUST be incorporated in the details element of the fault element in the SOAP message when the Service encounters an internal error when processing a particular function call.

Attributes of the internalServerError type (aka element)

This element has no attributes.

Content options for the internalServerError type (aka element)

The internalServerError element MAY contain the following elements:

  • exceptionMessage - This element contains the specific message associated to this exception type. This element is a REQUIRED element that MUST only occur once inside the internalServerError element. The data type of this element is String. The only allowed value for this element is: The Service encountered an internal server error when processing this function call.
  • exceptionTrace - This element contains the specific stack trace associated to this exception type. This element is a REQUIRED element that MUST only occur once inside the internalServerError element. The data type of this element is String. There are no restrictions on the lexical or value spaces of this element. The Service MUST however incorporate the specific stack trace in the value of this element.

6.3. The invalidParameter type

This section is normative.

This type MUST be incorporated in the details element of the fault element in the SOAP message when the Service deems that an invalid parameter was passed into the particular function call.

Attributes of the invalidParameter type (aka element)

This element has no attributes.

Content options for the invalidParameter type (aka element)

The invalidParameter element MUST contain the following elements:

  • exceptionMessage - This element contains the specific message associated to this exception type. This element is a REQUIRED element that MUST only occur once inside the invalidParameter element. The data type of this element is String. The only allowed value for this element is: An invalid parameter was passed into the function call.
  • exceptionTrace - This element contains the specific stack trace associated to this exception type. This element is a REQUIRED element that MUST only occur once inside the invalidParameter element. The data type of this element is String. There are no restrictions on the lexical or value spaces of this element. The Service MUST however incorporate the specific stack trace in the value of this element.

6.4. The notIssued type

This section is normative.

This type MUST be incorporated in the details element of the fault element in the SOAP message when the Reading System requests content that has not been issued.

Attributes of the notIssued type (aka element)

This element has no attributes.

Content options for the notIssued type (aka element)

The notIssued element MAY contain the following elements:

  • exceptionMessage - This element contains the specific message associated to this exception type. This element is a REQUIRED element that MUST only occur once inside the notIssued element. The data type of this element is String. The only allowed value for this element is: The content that was requested has not been issued.
  • exceptionTrace - This element contains the specific stack trace associated to this exception type. This element is a REQUIRED element that MUST only occur once inside the notIssued element. The data type of this element is String. There are no restrictions on the lexical or value spaces of this element. The Service MUST however incorporate the specific stack trace in the value of this element.

6.5. The serviceBusy type

This section is normative.

This type MUST be incorporated in the details element of the fault element in the SOAP message when the Service is busy.

Attributes of the serviceBusy type (aka element)

This element has no attributes.

Content options for the serviceBusy type (aka element)

The serviceBusy element MAY contain the following elements:

  • exceptionMessage - This element contains the specific message associated to this exception type. This element is a REQUIRED element that MUST only occur once inside the serviceBusy element. The data type of this element is String. The only allowed value for this element is: The Service is busy.
  • exceptionTrace - This element contains the specific stack trace associated to this exception type. This element is a REQUIRED element that MUST only occur once inside the serviceBusy element. The data type of this element is String. There are no restrictions on the lexical or value spaces of this element. The Service MUST however incorporate the specific stack trace in the value of this element.
